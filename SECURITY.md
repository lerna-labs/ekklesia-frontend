# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it privately rather than filing a public issue.

Email either of the maintainers directly:

- adam@crypto2099.io
- mad@madorkestra.com

Please include as much detail as you can: the affected component, steps to reproduce, and potential impact. This helps us triage and respond quickly.

**Do not file security vulnerabilities as public GitHub issues.** Public issues are visible to everyone, including potential attackers, before a fix is available.

## What to Expect

We will acknowledge your report within **5 business days**. From there we'll work with you to understand the issue, develop a fix, and coordinate disclosure.
